Though I have to balance my life and programming.And yet I need face the limit of terminal.I do a lot of experiment to ensure that checking combo key at a headlessconsole enviroment is possible. So that we could use complex combo key likeC-S-s to "save as" file.But I failed. I cannot reach it at a Unix-like system(I have tried on both Linux and FreeBSD) which has nographics interface.As we know that one of the aim of Folic is 100% terminal-friendly.It seems that my project will not have progress any more...But, after thinking for days. I think that I still need update this project.We cannot check complex combo key, but we still have chance to implementthe functions!The most important part of this update is Selection Area. Though you cannotuse S+[Arrow Keys] like what you usually do in other complete editors (Becausethe technical limit above-mentioned...). You can still use F2 key (I admit thatit looks strange) to do it. I used to try C-Space to achive it like Emacs,however it didn't work. So I just select F2. You can use C-c C-v C-x too.If you are a Mac user. I apology for the key set.And also. Now you don't need to worry about exiting Folic or open new filewithout saving current file. I just added y-n-cancel function, which is smallbut very useful. :)Now the editor will check the newline of file, instead of assume any filehas LF. Though you still need to change the newline of the editor with othertool. I apology :(Anyway, that's all. Enjoy the crusty editor!
